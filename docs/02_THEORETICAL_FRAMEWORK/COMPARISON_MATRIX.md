## 2. 语义模型对比

### 2.1 三种语义模型详细对比

| 维度 | 操作语义 | 指称语义 | 公理语义 |
|------|---------|---------|---------|
| **数学基础** | 状态转换系统 | 域理论 | 霍尔逻辑 |
| **核心思想** | 如何执行 | 意义是什么 | 满足什么性质 |
| **表示形式** | σ' = δ(σ, e) | ⟦e⟧: Env → Val | {P} C {Q} |
| **优点** | 直观易懂 | 抽象程度高 | 便于验证 |
| **缺点** | 细节过多 | 抽象难理解 | 需要逻辑基础 |
| **工具支持** | 高 | 中 | 高 |
| **OTLP应用** | 状态管理 | 行为建模 | 正确性证明 |

### 2.2 代码示例对比

#### 操作语义示例

```rust
// 操作语义: 明确的状态转换
struct State {
    value: i32,
}

impl State {
    fn transition(&mut self, operation: Op) {
        match operation {
            Op::Inc => self.value += 1,
            Op::Dec => self.value -= 1,
        }
    }
}

// 性能: O(1)
// 内存: O(1)
```

#### 指称语义示例

```rust
// 指称语义: 从语法到语义的映射
trait Denotation<T> {
    fn meaning(&self) -> T;
}

impl Denotation<i32> for Expr {
    fn meaning(&self) -> i32 {
        match self {
            Expr::Num(n) => *n,
            Expr::Add(l, r) => l.meaning() + r.meaning(),
        }
    }
}

// 性能: O(n) - n为表达式深度
// 内存: O(n) - 递归栈
```

#### 公理语义示例

```rust
// 公理语义: 前置后置条件
/// Precondition: x >= 0
/// Postcondition: result >= x
fn increment(x: i32) -> i32 {
    assert!(x >= 0);
    let result = x + 1;
    assert!(result >= x);
    result
}

// 性能: O(1) + 断言开销
// 内存: O(1)
```
